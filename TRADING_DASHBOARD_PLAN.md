## Executive Summary

This document outlines the architecture and implementation plan for an **end-to-end quantitative trading dashboard** specifically designed for oil markets. The system will integrate market intelligence, quantitative research, model deployment, signal generation, risk management, and trade execution into a unified platform.

**Target Users:** Oil traders, quantitative analysts, risk managers  
**Asset Classes:** Crude Oil (WTI, Brent), Refined Products, Spreads, Options  
**Deployment:** Cloud-native with hybrid edge capabilities

### 3.3 Research Database Schema

```sql
-- Core research tables (conceptual)

-- Strategy metadata
CREATE TABLE strategies (
    strategy_id UUID PRIMARY KEY,
    name VARCHAR(255),
    description TEXT,
    asset_class VARCHAR(50),
    strategy_type VARCHAR(50),  -- momentum, mean_reversion, arbitrage, etc.
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    status VARCHAR(20),  -- research, paper_trading, live, retired
    author VARCHAR(100)
);

-- Backtest results
CREATE TABLE backtest_runs (
    run_id UUID PRIMARY KEY,
    strategy_id UUID REFERENCES strategies,
    parameters JSONB,
    start_date DATE,
    end_date DATE,
    sharpe_ratio DECIMAL,
    max_drawdown DECIMAL,
    total_return DECIMAL,
    win_rate DECIMAL,
    num_trades INTEGER,
    metrics JSONB,  -- Full metrics JSON
    created_at TIMESTAMP
);

-- Signal archive
CREATE TABLE historical_signals (
    signal_id UUID PRIMARY KEY,
    strategy_id UUID REFERENCES strategies,
    timestamp TIMESTAMP,
    instrument VARCHAR(50),
    direction INTEGER,  -- -1, 0, 1
    confidence DECIMAL,
    features JSONB,
    metadata JSONB
);
```
